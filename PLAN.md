# keySwitcher — план реализации

Нативная open-source утилита для macOS, аналог Punto Switcher: конвертация набранного текста между раскладками (например `ghbdtn` → `привет`) по горячей клавише. Живёт в меню-баре (menu bar / «трей»), работает постоянно в фоне.

---

## 1. Цели и объём

### MVP (обязательно)
- Конвертация **последнего набранного слова / фразы** по хоткею (по умолчанию — `Option+Shift+S`; в настройках пресеты, включая двойной `Shift`, и произвольная комбинация).
- Конвертация **выделенного текста** по отдельному хоткею.
- Автоматическое **переключение системной раскладки** после конвертации.
- **Пара раскладок берётся из системы**: если установлено ровно 2 источника ввода — работаем с ними автоматически; если больше — пользователь выбирает пару в настройках (до выбора конвертация неактивна, иконка подсказывает открыть настройки).
- Иконка в меню-баре: текущая раскладка, меню (пауза, настройки, выход).
- Окно настроек: хоткеи, автозапуск, выбор пары раскладок.
- Автозапуск при логине.

### Пост-MVP (потом, отдельными итерациями)
- Автопереключение по словарю (как в Punto: детект «не той» раскладки на лету) — самая сложная часть, осознанно выносим за MVP.
- Пользовательские правила-исключения, список приложений-исключений.
- Смена регистра выделенного (`привет` → `ПРИВЕТ`), транслитерация.
- Звуковая индикация, история буфера.

### Не делаем
- Дневник/кейлоггинг с записью на диск (репутационно токсично для open source).
- Поддержка macOS < 13 (Ventura) — упрощает автозапуск и SwiftUI.

## 2. Ключевые технологии macOS

Это фундамент, вокруг которого строится всё приложение:

| Задача | API | Заметки |
|---|---|---|
| Перехват клавиатуры глобально | `CGEventTap` (CoreGraphics) | Требует права **Accessibility** (и Input Monitoring). Один tap на `keyDown` + слушаем клики мыши для сброса буфера |
| Хоткеи | тот же `CGEventTap` | Carbon `RegisterEventHotKey` не умеет «двойной Shift» (это flagsChanged), поэтому хоткеи детектим сами в tap-е: и обычные комбо (`Option+Shift+S`), и modifier-only |
| Текущая раскладка и её смена | `TISCopyCurrentKeyboardInputSource`, `TISSelectInputSource` (Carbon/HIToolbox, работает и в Swift) | Список источников: `TISCreateInputSourceList` |
| keycode → символ в конкретной раскладке | `UCKeyTranslate` + `kTISPropertyUnicodeKeyLayoutData` | Даёт честную конвертацию для **любых** пар раскладок, а не только RU/EN по захардкоженной таблице |
| Синтез ввода (стереть и перепечатать) | `CGEvent(keyboardEventSource:)` + `CGEventKeyboardSetUnicodeString` + `CGEvent.post` | Unicode-string-событие печатает текст независимо от активной раскладки |
| Работа с выделенным текстом | Вариант А: Accessibility API (`AXUIElement`, `kAXSelectedTextAttribute`) — чисто, но работает не везде (не в Chrome/Electron без флагов). Вариант Б (fallback): симуляция `Cmd+C` → правка Pasteboard → `Cmd+V` → восстановление буфера обмена | Делаем А с автоматическим fallback на Б |
| Иконка в меню-баре | `NSStatusItem` (AppKit) | `LSUIElement = YES` в Info.plist — без иконки в Dock |
| Настройки UI | SwiftUI-окно внутри AppKit-приложения | `Settings`-сцена или отдельное `NSWindow` |
| Автозапуск | `SMAppService.mainApp.register()` (macOS 13+) | Одна строчка, без helper-таргетов |
| Хранение настроек | `UserDefaults` (+ `@AppStorage`) | Хоткеи сериализуем как keycode+modifiers |
| Сброс буфера при смене окна | `NSWorkspace.didActivateApplicationNotification` | Плюс сброс по кликам мыши, Enter, Esc, стрелкам |

**Главный подводный камень:** `CGEventTap` без включённого Accessibility молча не создаётся. Нужен явный онбординг: проверка `AXIsProcessTrustedWithOptions`, экран с кнопкой «Открыть настройки системы», поллинг до выдачи права. После пересборки в debug право иногда «слетает» (меняется подпись) — в README для разработчиков это надо описать.

### Модули и их контракты

1. **`EventTapManager`** — владеет `CGEventTap`. На каждый `keyDown`:
   - отдаёт событие `HotkeyDetector`-у (детект хоткеев: обычные комбинации по `keyDown`+флаги, двойной Shift — по `flagsChanged` с таймингом < 300 мс);
   - если не хоткей — отдаёт в `KeyBuffer` (keycode + флаги + активная раскладка на момент нажатия);
   - клики мыши / Enter / Esc / стрелки / Cmd-комбо → сигнал сброса буфера.
   - Работает на отдельном `CFRunLoop`/потоке, коллбэк максимально лёгкий (иначе macOS отключает tap по таймауту — нужен обработчик `kCGEventTapDisabledByTimeout` с ре-энейблом).

2. **`KeyBuffer`** — кольцевой буфер последних N (~100) нажатий: `(keycode, modifiers, inputSourceID)`. Умеет: выдать «последнее слово» (до пробела/сброса), «последнюю строку», обработать Backspace (выталкивание). **Хранится только в памяти, никогда не пишется на диск.**

3. **`LayoutConverter`** — два режима:
   - *По keycode* (основной): буфер хранит keycodes → прогоняем их через `UCKeyTranslate` с раскладкой-целью. Работает для любых пар раскладок без таблиц.
   - *По символам* (для выделенного текста, где keycodes неизвестны): обратный маппинг символ → keycode через таблицу, построенную из `UCKeyTranslate` при старте для каждой установленной раскладки, затем прямой прогон в целевую. Для пары, где символа нет в раскладке — оставляем как есть.

4. **`TextReplacer`** — стратегии замены набранного:
   - синтез N событий Backspace (по длине конвертируемого куска), затем печать результата одним `CGEventKeyboardSetUnicodeString` (чанками ≤ 20 символов — ограничение API);
   - важно: **временно отключать обработку собственных синтетических событий** (помечать их через `CGEventSetIntegerValueField` кастомным userData и игнорировать в tap-е), иначе бесконечный цикл.

5. **`SelectionConverter`** — конвертация выделенного: AX API → если элемент не отдаёт `kAXSelectedText`, fallback через Pasteboard (сохранить содержимое, Cmd+C, подождать changeCount, конвертировать, Cmd+V, восстановить буфер обмена с задержкой).

6. **`InputSourceManager`** — обёртка над TIS: список установленных раскладок (только keyboard layouts, без палитр эмодзи и т.п.), текущая, переключение, подписка на `kTISNotifySelectedKeyboardInputSourceChanged` (обновлять иконку в меню-баре). Логика пары: ровно 2 источника в системе → пара определяется автоматически; больше 2 → пара из `SettingsStore`, пока не выбрана — конвертация выключена. Реагирует на добавление/удаление раскладок в системе (сброс невалидной пары).

7. **`StatusBarController`** — `NSStatusItem` с иконкой/буквами текущей раскладки (EN/RU), меню: Convert last word (с хоткеем), Convert selection, Pause, Settings…, Launch at login, About, Quit.

8. **`SettingsStore` + `SettingsWindow`** — SwiftUI: вкладки General (автозапуск, пауза), Hotkeys (пресеты: `Option+Shift+S` — дефолт, двойной Shift, `Ctrl+Option+S`, «Break/Pause-стиль» + рекордер произвольной комбинации), Layouts (выбор пары из установленных в системе; секция видна только если раскладок > 2).

## 4. Ключевые сценарии (флоу)

### Сценарий 1: «Набрал не в той раскладке» (основной)
1. Пользователь печатает `ghbdtn vbh` в английской раскладке.
2. Каждое нажатие ложится в `KeyBuffer` с пометкой раскладки.
3. Хоткей (`Option+Shift+S`) → `HotkeyDetector` срабатывает.
4. Берём из буфера последнее слово (или всё с последнего сброса — настраивается; повторное нажатие хоткея расширяет: слово → вся фраза, как в Punto).
5. `LayoutConverter`: keycodes → символы в противоположной раскладке (`привет мир`).
6. `TextReplacer`: N × Backspace + печать результата.
7. `InputSourceManager.switch()` → системная раскладка теперь русская.
8. Буфер обновляется: то же слово, но с новой пометкой раскладки (чтобы повторный хоткей конвертировал обратно — операция должна быть обратимой).

### Сценарий 2: конвертация выделенного
1. Пользователь выделяет текст, жмёт второй хоткей.
2. AX / Pasteboard-fallback достаёт текст, `LayoutConverter` конвертирует посимвольно, заменяем, выделение по возможности восстанавливаем.

### Сценарий 3: первый запуск
1. Нет права Accessibility → окно онбординга с объяснением, зачем (и что ничего не пишем на диск), кнопка в System Settings → Privacy & Security → Accessibility.
2. Поллинг `AXIsProcessTrusted()` раз в секунду → как только выдали, стартуем tap и прячем окно.

## 6. Этапы реализации

### Этап 0 — Скелет (½ дня)
- XcodeGen-проект, `LSUIElement`, пустой `NSStatusItem` с меню Quit.
- `PermissionsService` + окно онбординга Accessibility.
- ✅ Критерий: иконка в меню-баре, приложение просит и получает право.

### Этап 1 — Перехват и буфер (1–2 дня)
- `EventTapManager` с ре-энейблом по таймауту, маркировка своих событий.
- `KeyBuffer` со сбросами (мышь, Enter, смена приложения, Backspace).
- Debug-вывод буфера в консоль.
- ✅ Критерий: в логе видно последнее набранное слово в любом приложении.

### Этап 2 — Конвертация и замена (2–3 дня) — сердце проекта
- `LayoutConverter` на `UCKeyTranslate` + юнит-тесты на пары RU↔EN (`ghbdtn`↔`привет`, пунктуация `;`↔`ж`, Shift-символы).
- `TextReplacer`: backspaces + Unicode-печать чанками.
- `InputSourceManager` (автопара при 2 раскладках, пара из настроек при > 2) + переключение раскладки после замены.
- Захардкоженный хоткей `Option+Shift+S` для теста.
- ✅ Критерий: `ghbdtn` + хоткей → `привет` в TextEdit, Safari, Telegram; повторный хоткей — обратно.

### Этап 3 — Хоткеи и меню-бар (1–2 дня)
- `HotkeyDetector`: произвольные комбинации (keyDown + флаги) и modifier-only «двойной Shift» (flagsChanged + тайминг) как альтернативный пресет.
- Иконка = текущая раскладка, реагирует на `kTISNotifySelectedKeyboardInputSourceChanged`.
- Полное меню, Pause.
- ✅ Критерий: `Option+Shift+S` и двойной Shift работают, иконка меняется при смене раскладки.

### Этап 4 — Выделенный текст (1–2 дня)
- AX-путь + Pasteboard-fallback с восстановлением буфера обмена.
- ✅ Критерий: конвертация выделения работает в TextEdit (AX) и в Chrome (fallback).

### Этап 5 — Настройки и полировка (2 дня)
- SwiftUI-настройки: пресеты хоткеев + рекордер, выбор пары раскладок (при > 2 в системе), автозапуск (`SMAppService`).
- Обработка краевых случаев: печать во время замены, длинные тексты, secure input fields (`IsSecureEventInputEnabled()` — в полях паролей буфер не ведём вообще).
- ✅ Критерий: MVP-фичелист закрыт.

### Этап 6 — Релиз (1 день + бюрократия)
- README (скриншоты, инструкция по Accessibility), MIT LICENSE.
- Пока Developer-аккаунт оформляется: релиз без нотаризации — инструкция про `xattr -d com.apple.quarantine` и сборку из исходников. Как только аккаунт готов — подпись Developer ID + нотаризация в CI (без неё Gatekeeper запугает пользователей).
- GitHub Actions: build + test + релизный `.dmg` (create-dmg), Homebrew cask.

**Итого MVP: ~8–11 дней чистой работы.**

## 7. Риски и краевые случаи

| Риск | Митигция |
|---|---|
| macOS отключает event tap по таймауту | Лёгкий коллбэк, обработка `tapDisabledByTimeout` → re-enable |
| Свои синтетические события попадают в свой же tap | Метка через `CGEventSetIntegerValueField(.eventSourceUserData)` |
| Secure input (пароли) | `IsSecureEventInputEnabled()` → буфер не пишем, иконку можно менять на «замок» |
| Backspace-замена в приложениях с автодополнением (адресная строка браузера) | Известная боль всех аналогов; частично лечится AX-заменой, где возможна |
| Право Accessibility слетает при пересборке | Задокументировать; стабильная подпись (dev certificate) решает |
| Нелатинские/неcyrillic-раскладки | `UCKeyTranslate`-подход работает для любых пар из коробки |
| Восприятие как кейлоггера | Буфер только в RAM, маленький, код открыт — явно проговорить в README |
| Chrome/Electron не отдают AX-выделение | Pasteboard-fallback |

## 9. Принятые решения

1. **Apple Developer аккаунт** — пока нет, будет оформлен. До этого: локальная подпись для разработки, релизы с инструкцией обхода Gatekeeper; нотаризация добавляется в CI после получения аккаунта.
2. **Раскладки** — берутся из системы. Ровно 2 установлены → пара автоматически; больше 2 → выбор пары в настройках (до выбора конвертация неактивна). Универсальная конвертация через `UCKeyTranslate`, без захардкоженных таблиц.
3. **Хоткей по умолчанию** — `Option+Shift+S`. В настройках пресеты (в т.ч. двойной Shift) + рекордер произвольной комбинации.
